## 1. Email Sequence Architecture — Complete Overhaul

### Old Approach
14 days of ICP-specific pitch emails. Every day was a hard sell targeted to the niche.

### New Approach: Two-Track System

The sequence is now split into two tracks — a **shared elevated track** and a **per-ICP spike** — assembled by a helper at runtime.

| Days | Track | Description |
|------|-------|-------------|
| 1–2 | Elevated (shared) | Dan Koe/Tom Bilyeau/Tim Denning voice. The Layer themes. Attention residue, architecture vs discipline. NO pitch, NO CTA. |
| 3–5 | ICP Spike (unique per ICP) | 3 hard niche-specific pitch emails. These are the only days with a direct ask. |
| 6–14 | Elevated continuation (shared) | Inbox trap, calendar problem, professor model, AI operating layer, social without the scroll, compounding architecture decisions, operator identity, soft close. |

This applies to ALL active ICPs. No ICP has a fully custom 14-day run anymore.

### New Files

| File | Purpose |
|------|---------|
| `icp/sequences/elevated.ts` | All shared elevated email prompts — days 1–2 and days 6–14 |
| `icp/sequences/build.ts` | `buildICPSequence(spike)` helper — assembles a full 14-day sequence by merging the shared elevated emails with the ICP-specific spike |

### ICP Sequence Files — Rewritten

Each ICP sequence file now only defines its 3-email spike (days 3–5). The rest is assembled by `buildICPSequence`.

| File | Spike Topics |
|------|-------------|
| `icp/sequences/med-spa.ts` | Site costing clients / content engine / 14-day close |
| `icp/sequences/claude-consulting.ts` | The gap / clients will ask about AI / what we build |
| `icp/sequences/agency.ts` | Owner-as-bottleneck / AI strategy positioning / infrastructure close |
| `icp/sequences/dental.ts` | Ranking for procedures not name / content without admin burden / close |
| `icp/sequences/chiropractic.ts` | Clinical day ends clean business doesn't / marketing without owner / close |

## 3. Bounce Detection Added

### What changed in `check-replies.ts`
A bounce detection pass now runs after the regular reply check. It:
1. Searches Gmail for mailer-daemon / delivery failure emails
2. Parses the bounced recipient addresses from those messages
3. Marks matching leads as `status='bounced'` in the DB

### What changed in `email-outreach.ts`
Bounced leads are now skipped during send. The pipeline will not attempt to email any lead with `status='bounced'`.

**Going forward:** Bounce handling is automatic. No manual cleanup required unless a lead is misclassified.

---

## 4. Strategic Direction — What the Sequence Is Designed to Do

The elevated track is not a sales sequence. It is a relationship track. The goal is to stay in the inbox indefinitely — build trust, establish authority, earn the read. The ICP spike (days 3–5) is the one hard ask. After that, the sequence returns to value delivery.

**Post-day-14 plan (infrastructure TBD):** Active leads (opened/clicked/replied) migrate to a newsletter list. The outreach pipeline hands them off; a newsletter system picks them up. This infrastructure does not exist yet — flag it when building.
